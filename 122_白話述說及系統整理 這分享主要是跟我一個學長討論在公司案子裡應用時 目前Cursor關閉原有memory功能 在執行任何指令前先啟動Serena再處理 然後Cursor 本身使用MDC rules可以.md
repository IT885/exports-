| 影響層面 | 詳細說明 |
| :--- | :--- |

| **從隱性到顯性** | 過去，Cursor 的「記憶」更多體現在它能理解整個程式碼庫的上下文[2]。現在，1.0 版的「Memories」功能將其變為一個**使用者可以主動管理和查閱**的具體功能，讓記憶變得更加透明和可控[1]。 |

總結來說，Cursor 並未關閉記憶功能，而是將其**從一個相對底層的上下文理解能力，升級為一個標準化、可管理的官方「Memories」功能**。這個轉變可能會讓一些依賴舊有機制建立的自訂工作流程（如您所設計的）需要調整，但對整體使用者而言，這是一次重要的功能升級與體驗優化。
