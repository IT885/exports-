### 对话 2

**问题**: 這也跟路由器一樣 有config沒錯吧

**回答**:
沒錯，**Cisco Catalyst 9800-80-K9 無線控制器**的操作方式和傳統 Cisco 路由器非常類似，同樣是透過 CLI（命令列介面）進行設定，並且有完整的 config（設定檔）管理機制。[1][2]

你可以像設定路由器一樣，進入特權模式（enable）、進入全域設定模式（configure terminal），然後設定各種參數，例如：

- 設定主機名稱（hostname）
- 設定管理介面 IP、VLAN、預設路由
- 設定 SSH、HTTPS、密碼與使用者帳號
- 設定無線相關參數（如 country code、AP 管理等）

設定完成後，也需要像路由器一樣，**儲存設定**到 NVRAM（例如用 `write memory` 或 `copy running-config startup-config`），確保重開機後設定不會遺失。[2]

如果你熟悉 Cisco 路由器的 CLI 操作，設定 C9800-80-K9 會非常直覺，基本上就是「有 config 沒錯」。[1][2]

根據您的描述和configuration，我的分析是這樣：
 
原因說明：
根據目前的設定與拓樸檢視，發現有以下情況：
•	HZ-13516 的 Gi1/4 與 HZ-13517 的 Gi1/3 被正確地設定為同一 REP segment（segment ID 11）之 trunk port。
•	HZ-13517 的 Gi1/10 被設定為 access VLAN 21 的 access port，但此 port 誤接至 Gi1/3，造成一條非預期的 loop path。
•	VLAN 21 的流量從 Gi1/10 傳入 Gi1/3，進入 REP 環狀拓樸，再從 Gi1/4 傳回，形成一個 Layer 2 迴圈。並產生以下 log
Jun 17 17:05:29.599: %SW_MATM-4-MACFLAP_NOTIF: Host 0080.4ce4.67fe in vlan 21 is flapping between port Gi1/4 and port Gi1/10
•	由於 Gi1/10 並未參與 REP segment，REP 機制無法得知這條多出來的 path，因此 無法主動封鎖此 loop。
•	最終導致 VLAN 21 的 broadcast/multicast 封包持續循環，產生 broadcast storm。
 
為何 REP 機制未能阻止此次事件：
REP 只能控制同一 segment 中的 port，對於未設定 REP 的 port（如 access port Gi1/10）並不會交換控制訊息，也無法監控其轉發狀況，因此無法封鎖這條錯誤的連線所造成的 loop。
 
改善建議：
1.	確保 REP trunk port 僅連接至其他參與同一 segment 的 REP port。
2.	避免將 REP port 接至 access port，特別是 access port 所屬 VLAN 也包含在 trunk VLAN 中的情況。
3.	建議進行 實體配線查核與標示清楚各 port 的用途，降低誤接風險。

原因說明：
根據目前的設定與拓樸檢視，發現有以下情況：
•	HZ-13516 的 Gi1/4 與 HZ-13517 的 Gi1/3 被正確地設定為同一 REP segment（segment ID 11）之 trunk port。
•	HZ-13517 的 Gi1/10 被設定為 access VLAN 21 的 access port，但此 port 誤接至 Gi1/3，造成一條非預期的 loop path。
•	VLAN 21 的流量從 Gi1/10 傳入 Gi1/3，進入 REP 環狀拓樸，再從 Gi1/4 傳回，形成一個 Layer 2 迴圈。並產生以下 log

•	由於 Gi1/10 並未參與 REP segment，REP 機制無法得知這條多出來的 path，因此 無法主動封鎖此 loop。
•	最終導致 VLAN 21 的 broadcast/multicast 封包持續循環，產生 broadcast storm。
 
為何 REP 機制未能阻止此次事件：
REP 只能控制同一 segment 中的 port，對於未設定 REP 的 port（如 access port Gi1/10）並不會交換控制訊息，也無法監控其轉發狀況，因此無法封鎖這條錯誤的連線所造成的 loop。
 
改善建議：
1.	確保 REP trunk port 僅連接至其他參與同一 segment 的 REP port。
2.	避免將 REP port 接至 access port，特別是 access port 所屬 VLAN 也包含在 trunk VLAN 中的情況。
