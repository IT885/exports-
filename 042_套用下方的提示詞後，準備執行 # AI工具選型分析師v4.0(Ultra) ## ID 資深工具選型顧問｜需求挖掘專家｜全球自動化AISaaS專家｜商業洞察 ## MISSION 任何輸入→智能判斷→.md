## 🤗 Hugging Face 計價方式詳解

### 三種服務模式

#### **1. 免費推理 API（Inference API - Free）**
- **價格**：完全免費
- **限制**：社群模型、有速率限制、無 SLA 保證
- **適合**：實驗、學習、小型專案
- **Pro 會員**：$9/月，提高速率限制和優先級[3][11][12][13]

#### **2. 推理端點（Inference Endpoints - Dedicated）**
- **計費方式**：按小時收費（硬體使用時間 × 算力成本）
- **起價**：$0.033/小時起
- **特點**：
  - 專屬 GPU 資源
  - 自動擴展
  - 企業級安全
  - 依選擇的硬體類型計費（CPU/GPU）
- **透明度問題**：無法事先知道具體模型使用哪種硬體，難以預估成本[11][14][3]

#### **3. 企業協作方案（Enterprise Hub）**
- **價格**：$20/月/用戶起
- **功能**：團隊協作、模型共享、審計日誌、企業標準[11]

### Hugging Face 的計價挑戰
⚠️ **成本不透明**：用戶反映難以追蹤實際 API 請求成本，因為不清楚每個模型使用的硬體配置和推理時間。[14]

#### 技巧 4：速率限制器（避免超支）

```python
import time
from collections import deque

class RateLimiter:
    def __init__(self, max_calls=60, time_window=60):
        """限制每分鐘最多 60 次請求（Google AI Studio 免費層）"""
        self.max_calls = max_calls
        self.time_window = time_window
        self.calls = deque()
    
    def allow_request(self):
        """檢查是否允許請求"""
        now = time.time()
        
        # 移除時間窗口外的記錄
        while self.calls and self.calls[0] < now - self.time_window:
            self.calls.popleft()
        
        # 檢查是否超過限制
        if len(self.calls) >= self.max_calls:
            wait_time = self.time_window - (now - self.calls[0])
            print(f"⚠️ 達到速率限制，等待 {wait_time:.1f} 秒")
            time.sleep(wait_time)
        
        # 記錄請求
        self.calls.append(now)
        return True

# 使用範例
limiter = RateLimiter(max_calls=60, time_window=60)

for i in range(100):
    limiter.allow_request()
    result = call_google_ai_studio(f"請求 {i}")
    print(result)
```

**好處**：避免觸發速率限制導致請求失敗[12]
