## 為什麼會發生這個問題？

這個問題通常是因為文本中包含的換行符（newline）觸發了模型的自動執行機制。大多數聊天界面將"Enter"鍵視為提交命令的信號，當您粘貼含有換行符的文本時，系統會誤認為您已完成輸入。[5]

請等待我的完整輸入，不要在我說"請開始分析"之前開始處理。以下是我將要粘貼的內容：

[在此粘貼您的內容]

請開始分析
```

在某些平台上，您可以使用代碼塊格式來粘貼內容：

這個技巧來自PowerShell但也適用於某些AI平台：[5]

- **某些編輯器**：可以先在本地編輯器中編寫，然後一次性粘貼

1. **預估分段數量**：
   ```

   ```

2. **標準化分段標記**：
   ```
   每段開頭添加「第N段，共M段」

   ```

3. **分段內容預告**：
   ```
   每段結尾提示下一段將要討論的內容
   如「下一段將討論：1. XXX 2. YYY 3. ZZZ」
   ```

4. **最後一段標記**：
   ```
   最後一段結束時明確標示「分析完成」
   ```

這種方法能夠提前告知用戶總段數、當前進度，以及後續是否還有內容，大大改善使用體驗。

1. 評估內容總量並預測所需段數
2. 跟踪tokens消耗，在接近限制前主動添加分段標記
3. 在每段結尾保留上下文連貫性的提示
