# Claude MCP與GitHub Copilot訂閱關係解析

## Claude的MCP是什麼

Model Context Protocol (MCP)是Anthropic開發的開源標準，讓Claude等AI模型能與外部工具和數據源進行互動。它就像AI的「USB-C接口」，標準化了AI與外部系統的連接方式。通過MCP，Claude可以：[7][8][9]

- 實時上網搜尋最新資訊[10]
- 進行網頁爬蟲收集數據[10]
- 操作瀏覽器完成任務[8]
- 管理本地文件[10]
- 建立個人知識庫並記憶對話內容[10]

## GitHub Copilot訂閱計劃

GitHub提供三種不同的Copilot訂閱計劃：[2]

**GitHub Copilot Free**
- 每月限制2,000次代碼補全
- 每月限制50次聊天請求
- 免費，但功能受限

**GitHub Copilot Pro**
- 無限代碼補全
- 每月300次高級請求
- 月費$10美元或年費$100美元

**GitHub Copilot Pro+**
- 所有Pro功能
- 完整訪問所有可用模型
- 每月1,500次高級請求
- 月費$39美元或年費$390美元

## AI模型與Copilot的關係

**Claude模型在Copilot中的可用性**：
- Claude 3.5 Sonnet已可在GitHub Copilot中使用，通過Amazon Bedrock提供服務[4][6]
- Claude 3.7 Sonnet也已推出，但不適用於Copilot Free用戶[4]

**OpenAI模型在Copilot中的可用性**：
- GPT-4.1已於2025年4月推出，並適用於所有Copilot計劃，包括Free用戶[5]
- 該模型較GPT-4o有明顯提升，特別是在編碼和長上下文理解方面[5]

## Agent模式與MCP的關係

Agent模式是Copilot的一項強大功能，可以：
- 自主規劃步驟、選擇正確的文件、運行工具或終端命令[3]
- 處理跨項目的複雜任務並保持會話上下文[3]
- 自動應用編輯，而非每步都等待明確許可[3]

但是，**根據搜索結果，GitHub Copilot的Claude整合與直接使用Claude的MCP功能似乎是分開的系統**。搜索結果中沒有明確提到Copilot訂閱能否直接使用Claude的MCP功能。

根據資料，Anthropic在2025年5月1日宣佈了遠程MCP支持（也稱為Integrations），讓用戶可以將Claude AI連接到雲託管服務器。但這似乎是Claude自己的服務特性，而非通過GitHub Copilot提供。[10]

## 結論

1. 如果您想使用完整的Claude MCP功能，可能需要直接透過Anthropic的Claude服務。

2. GitHub Copilot確實提供了Claude 3.5和3.7模型（後者不適用於Free用戶），以及GPT-4.1模型。

3. Copilot的Agent模式功能強大，但與Claude的MCP是不同的技術，雖然兩者都能提升AI的能力。

若要確認最新的整合狀態，建議查看GitHub和Anthropic的官方文檔，因為AI服務整合關係可能會隨著新合作而變化。

# MCP客戶端實用指南：配合GitHub Copilot環境使用

## 在VS Code中使用MCP與Copilot

既然您已有GitHub Copilot訂閱，最簡單的方式就是直接在VS Code中使用其Agent模式結合MCP功能：

**設置步驟：**
1. 更新VS Code到最新版本
2. 確認已啟用GitHub Copilot的Agent模式
3. 打開命令面板（Ctrl+Shift+P）並選擇「**MCP: Add Server**」[2]
4. 設置您想使用的MCP服務器（以下有具體例子）

**實用例子：**
設置一個網頁爬蟲MCP服務器：
```json
// .vscode/mcp.json
"servers": {
  "fetch": {
    "type": "stdio",
    "command": "uvx",
    "args": ["mcp-server-fetch"]
  }
}
```
這樣設置後，您就可以在Copilot聊天中直接請求：「幫我獲取最新的台灣股市資訊並分析趨勢」，Copilot會自動調用fetch工具獲取最新資料。[3][5]

## 使用MCP CLI Client無需Claude API

MCP CLI Client讓您可以在命令行使用MCP功能：

**設置步驟：**
1. 使用npm安裝MCP CLI工具：`npm install -g mcp-cli`
2. 設置一個本地設定檔（例如`~/.mcp-config.json`）
3. 在配置文件中使用現有模型API（您可以使用Copilot提供的模型而非Claude API）[5]

**實用例子：**
```bash
# 在終端中使用MCP CLI調用工具
mcp-cli "幫我分析這個git倉庫的提交歷史" --tool git-analyzer
```

## 設置Tome桌面應用

Tome是一個跨平台桌面應用，可與MCP服務器協作：

**設置步驟：**
1. 從GitHub下載並安裝Tome
2. 配置使用與VS Code相同的MCP服務器（它可以自動發現VS Code已配置的服務器）[2]
3. 使用您現有的GitHub認證登入（無需Claude API）

**實用例子：**
您可以在Tome中發送請求：「創建一個新的React組件，連接到我的後端API，並處理用戶驗證」。Tome會使用您配置的MCP服務器來協助完成這個任務。[7]

## 搭建MCP Simple Slackbot

如果您想在團隊協作中使用MCP功能：

**設置步驟：**
1. 創建一個Slack應用（按搜索結果中的指引）[4]
2. 克隆並設置MCP Simple Slackbot
3. 在`.env`文件中配置您已有的API密鑰（可以用OpenAI的而非Claude的）[4]

**實用例子：**
團隊成員可以在Slack中@機器人：「幫我檢查最新代碼中的bug，分析並提供修復建議」。機器人會使用配置好的MCP工具來執行這個任務。[4]

## 使用開源的VSCode擴展增強功能

**Copilot-MCP擴展**讓管理MCP服務器變得更容易：

**設置步驟：**
1. 在VS Code市場安裝Copilot-MCP擴展
2. 通過擴展UI發現並安裝開源MCP服務器
3. 無縫集成到您的Copilot工作流中[6]

**實用例子：**
安裝後，您可以在VS Code活動欄中找到「MCP Servers」按鈕，輕鬆管理、搜索和發現新的開源MCP服務器。然後在Copilot聊天中直接使用這些工具。[6]

# MCP工具實用指南與實現方法

## 什麼是MCP（Model Context Protocol）

MCP是由Anthropic開源的協議，目的是讓AI助手能夠與外部系統互動，例如存取檔案系統、網路搜尋、第三方服務等。它可以被視為「AI應用的USB接口」，提供標準化的通訊方式。[5]

MCP系統由三個核心組件組成：
- **MCP主機（客戶端）**：如Claude Desktop、Cursor等AI工具
- **MCP客戶端**：處理AI模型與服務器間的通訊
- **MCP伺服器**：提供特定功能的輕量級程式[1][5]

## 環境準備與基本設置

### 必備條件
- Python 3.10+（用於Python實作）
- Node.js（用於Node.js實作）
- 支援MCP的AI客戶端（如Claude Desktop、Cursor、Windsurf等）[2]

### 基本配置步驟

**1. 選擇MCP客戶端**
根據您的需求選擇合適的MCP客戶端，如Cursor、Claude Desktop等。[1]

**2. 配置MCP伺服器**
在客戶端中設置MCP伺服器配置，通常為JSON格式：

```json
"mcpServers": {
  "伺服器名稱": {
    "command": "執行命令",
    "args": ["參數"],
    "env": { "環境變數": "值" }
  }
}
```

對於Cursor，可在以下位置設置：
- 專案特定：`.cursor/mcp.json`
- 全局設置：`~/.cursor/mcp.json`[1]

## 實作MCP伺服器的方法

### 方法一：使用FastMCP建立Python伺服器

**步驟一：安裝必要套件**
```bash

uv python pin 3.10

source .venv/bin/activate
uv add "mcp[cli]" fastmcp requests
```

**步驟二：建立MCP伺服器代碼**
建立`main.py`文件：

```python
import requests
from fastmcp import FastMCP

# 建立MCP伺服器
mcp = FastMCP('Demo')

@mcp.tool()
def fetch_buffett_indicator():
    url = 'https://buffettindicator.net/wp-content/themes/flashmag/data/csvjson.json'
    response = requests.get(url, headers={'User-Agent': 'Mozilla/5.0'})
    if response.status_code == 200:
        return response.json()[-1]
    return None

if __name__ == '__main__':
    # 初始化並運行伺服器
    mcp.run(transport='stdio')
```

## Windows中的NPX命令問題

您嘗試使用`npx`命令時收到了PowerShell錯誤。這通常是因為PowerShell的執行安全政策限制。建議嘗試以下方法：

1. **使用命令提示符(CMD)代替PowerShell**：
   打開CMD並嘗試相同的NPX命令。[8]

2. **或者在PowerShell中修改執行策略**：
   ```powershell
   Set-ExecutionPolicy -Scope CurrentUser -ExecutionPolicy RemoteSigned
   ```
   然後嘗試使用NPX命令。[6][8]

2. **使用uvx運行MCP服務器**：
   ```powershell

   ```
   
3. **在VS Code配置中使用uvx**：
   ```json

       "type": "stdio",

     }
   }
   ```
