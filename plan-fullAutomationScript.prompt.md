自動化主題彙整腳本計畫

1. **輸入**
   - 指定一個資料夾，內含以數字前綴命名的 Markdown 檔（例如 `009_...md`）。
   - 可選地允許傳入特定檔案或模式。

2. **掃描**
   - 逐檔讀取並擷取文本內容。
   - 使用 LLM 或正則 heuristics 辨識：
     * 主題
     * 子主題
     * 關鍵字列表
     * 任何額外 meta，例如客戶、時間範圍。
   - 將結果儲存為類似下列結構：
     ```json
     {
       "009": {"topic": "對話整理機制", "subtopic": "郵件分析", "keywords": ["Notion","郵件","分類"]},
       ...
     }
     ```

3. **分組**
   - 依主題/子主題彙整檔案。
   - 產生 `groups[topic][subtopic] = [ids...]`。

4. **合併**
   - 對於每個主題：
     * 建立一個 Markdown 文件，前言區包含主題、子題、來源檔案列表、關鍵字。
     * 對每個子題，按照數字前綴排序檔案並合併內容。
     * 檢測並移除重複段落（例如重複的工具建議）。
     * 保留程式碼區塊與未解決問題。
     * 當同一子題存在多個版本時，可插入「演進過程」標題。

5. **輸出**
   - 將合併後的文件存到 `./processed/`，檔名如 `{topic}_完整版.md`。
   - 可選輸出分組結果的 JSON 日誌。

6. **驗證**
   - 提供指令執行抽樣準確度檢查：顯示隨機樣本的主題/子題判定。
   - 支援手動覆寫或校正。

7. **實作細節**
   - 語言：Python 腳本（未來可轉 PowerShell）。
   - 相依：OpenAI API 或本地 LLM、`python-frontmatter` 用於 YAML。
   - CLI 介面：`python aggregate.py --input exports --output processed`。

8. **後續步驟**
   - 原型主題識別功能。
   - 在真實檔案子集上測試。
   - 迭代分類 heuristics。
   - 完成合併邏輯並測試輸出格式。


此計畫將作為開發完整自動化腳本的基礎。
