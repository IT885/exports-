# Cisco Nexus Dashboard 節點重新部署系統說明

## 系統概述

這個系統是 **Cisco Nexus Dashboard (ND) 的節點重新部署流程**。Nexus Dashboard 是思科的統一管理平台，用於管理資料中心網路基礎設施。這份文件展示的是如何對 ND-NODE-L4 層級的節點進行重新映像檔安裝。[1]

## 網路架構分析

系統包含三個主要的網路層級：

| 網路類型 | 功能說明 | 顏色標示 |
|---------|----------|----------|
| **CIMC 網路** | 硬體管理網路，用於遠端管理伺服器硬體 | 藍色 |
| **Management Network** | 管理網路，用於系統管理和控制 | 紅色 |
| **Data Network** | 資料網路，處理實際的網路流量 | 黃色 |

## 部署流程系統化分析

### 第一階段：映像檔準備與掛載

1. **HTTP 伺服器設置**[1]
   - 在 HTTP 伺服器上準備 ND 映像檔 (`nd-dk9.3.1.1l.iso`)
   - 伺服器位址：`172.18.2.238:10080`

2. **CIMC 遠端掛載**[1]
   - 透過 SSH 或 Console 連接到 CIMC
   - 使用 `scope vmedia` 命令進入虛擬媒體管理
   - 執行 `map-www image` 命令掛載遠端映像檔

### 第二階段：系統重啟與網路配置

1. **硬體重啟**[1]
   - 使用 `scope chassis` 進入機箱管理
   - 執行 `power cycle` 重新啟動伺服器

2. **網路介面配置**[1]
   - 連接到主機 (`connect host`)
   - 選擇網路配置方式（DHCP 或靜態）
   - 配置適當的網路介面（如 `enp1s0f1`）

### 第三階段：叢集擴展

1. **首節點特殊處理**[1]
   - 第一個節點需要特殊的初始化步驟
   - 建立叢集的基礎架構

2. **後續節點加入**[1]
   - 添加其他節點到既有叢集
   - 進行最終配置

## 系統特點與技術要求

### 硬體需求
- **UCS 伺服器**：支援 CIMC 管理的思科統一運算系統
- **多網路介面**：需要管理、資料等多個網路連接

### 軟體需求
- **Nexus Dashboard 映像檔**：特定版本的作業系統映像
- **網路連通性**：確保各網路層級的連接正常

### 部署考量
- **網路隔離**：不同功能的網路需要適當隔離
- **高可用性**：多節點叢集提供冗餘保護
- **擴展性**：支援後續節點的動態加入

## 總結

這個系統本質上是一個**企業級網路管理平台的部署自動化流程**。它通過標準化的步驟，將思科 Nexus Dashboard 軟體部署到多個硬體節點上，形成一個統一的管理叢集。整個流程強調網路的分層管理、硬體的遠端控制，以及叢集的漸進式建構，確保在資料中心環境中能夠穩定可靠地運行。[1]

## 流程確認 ─ 一張圖就能理解的「先連網、再報到、最後集中下發」

| 階段 | 主要動作 | 關鍵重點 | 為什麼必要 |
|------|----------|----------|-------------|
| **1. 基礎網路設定** | -  在 CIMC 或安裝精靈裡為 Management 與 Data 介面設定 IP<br>-  指定 **永久 IP**（Permanent / External Service IP） | 節點要能先「看得到網路」才能下載映像檔並向叢集註冊[1] | 沒網就無法掛載 ISO、也無法與中控通訊 |
| **2. 節點報到（Join Cluster）** | -  第一台機器組成 Nexus Dashboard 叢集<br>-  後續節點開機後自動偵測並加入 | 節點加入時會把自己的狀態上報，中控台即時顯示健康度[1] | 中控必須知道有哪些成員、各自健康狀態 |
| **3. 中控台集中管理** | -  管理員透過瀏覽器登入 ND GUI<br>-  把交換器／Fabric 新增為「站點」<br>-  以範本或工作流程一次下發設定 | ND 提供單一入口與批次佈署功能，減少逐台登入[2][3] | 大幅降低人工作業量與設定錯誤 |
| **4. 後續維運** | -  在 GUI 監控告警、流量、端點<br>-  需要變更時於 GUI 編輯範本 → 按 *Deploy* | 所有裝置收到更新並自動套用[4] | 持續營運、不停機升級 |

### 1. 為什麼「先連網」是硬性規定？
- 安裝流程會要求你立刻為 **管理網路 (mgmt0) 與資料網路** 指定 IP、閘道與 VLAN；沒有這些資訊就無法繼續。[1]
- 固定的 **永久 IP** 讓後面的服務（例如 Fabric Controller、Insights）能穩定對外提供 API 與資料。[4]

### 2. 報到機制怎麼運作？
1. 節點透過已設定好的管理網路與第一台 ND 交換 *cluster token*。  
2. 中控自動把新節點加到叢集並同步憑證、使用者與服務。  
3. 在 GUI 的 **Admin Console → Infrastructure** 就能看見新節點狀態（Up/Down、磁碟、CPU）。

### 3. 從中控台批次修改設定的典型流程
1. **Sites** 頁面把新交換器／Fabric 加入管理。  
2. 在 **Services → NDFC (Fabric Controller)** 選擇範本，編輯 VLAN、BGP、VXLAN 等參數。  
3. 點 **Deploy**，選擇要套用的交換器清單。  
4. ND 會一台台下發組態、即時回報成功／失敗；若有錯誤可點擊日誌調整後重新佈署。

### 4. 一句話總結
是的——**先讓每台機器能上網並向 Nexus Dashboard 報到，之後就能在中控 GUI 裡統一改設定，再一次佈署到多台設備**。整個流程正是為了避免「逐台 SSH 手動敲指令」的繁瑣與風險。

## **核心管理功能**

### **1. 韌體升級管理**
- **統一升級介面**：透過 Operations > Firmware Management 來管理所有設備的韌體升級[1]
- **批次升級**：可以同時對多台設備進行韌體更新
- **升級監控**：即時查看升級進度和狀態
- **版本管理**：支援多個韌體版本的儲存和管理

### **2. 設定檔管理與部署**
- **範本化管理**：建立標準化的設定範本[2]
- **批次設定部署**：選擇目標設備後一次性下發設定
- **設定驗證**：部署前驗證設定的正確性
- **即時設定修改**：透過 GUI 直接修改並推送到設備

## **統一平台服務**

### **3. 網路監控與分析**
| 監控類型 | 功能說明 |
|---------|----------|
| **即時監控** | 24小時監控網路狀態和流量[3] |
| **效能分析** | 深度分析網路效能指標[4] |
| **故障預警** | 主動偵測並告警潛在問題 |
| **能源監控** | 監控交換器設備的能源使用情況[5] |

### **4. 多站點管理**
- **站點統一管理**：從單一介面管理多個資料中心[3]
- **跨雲管理**：支援多雲環境的網路管理[6]
- **全域視圖**：提供整體網路架構的單一檢視畫面

### **5. 服務生命週期管理**
- **服務安裝**：從 App Store 下載並安裝各種網路服務[4]
- **服務升級**：管理已安裝服務的版本升級
- **服務啟用/停用**：彈性控制服務的運行狀態
- **資源配置**：根據需求調整服務的運算資源

### **6. 自動化與編排**

- **高可用性**：叢集模式確保管理平台的高可用性

## **實際操作流程**

### **韌體升級典型流程**
1. **上傳韌體映像檔**：將新版韌體上傳到 Nexus Dashboard
2. **選擇目標設備**：在 GUI 中選擇要升級的交換器
3. **排程升級時間**：設定維護時段進行升級
4. **監控升級進度**：即時查看每台設備的升級狀態
5. **驗證升級結果**：確認升級成功並檢查設備狀態

### **設定檔修改典型流程**
1. **選擇管理範本**：選擇要修改的設定範本
2. **編輯參數**：在 GUI 中修改 VLAN、BGP、VXLAN 等參數[7]
3. **驗證設定**：系統自動驗證設定的合理性
4. **選擇部署目標**：選擇要套用的交換器清單
5. **執行部署**：點擊 Deploy 按鈕推送設定到設備

# 網路專有名詞白話解釋

好的，我來為你白話解釋那些專有名詞！

## **FHRP (First Hop Redundancy Protocol) - 第一跳冗餘協定**

### **白話解釋：網路的「備胎系統」**
**就像汽車的備胎一樣**，當主要的網路閘道壞掉時，備用的閘道會立刻接手。[1][2]

#### **實際運作方式**
- **平常狀態**：兩台路由器都存在，但只有一台在工作（叫做 Active）
- **故障時**：主要路由器掛掉，備用路由器（Standby）立刻接手
- **對用戶來說**：完全感覺不到切換，網路不會斷線[1]

#### **生活化比喻**
```
就像電梯有兩台，平常只開一台
如果正在用的電梯壞了，另一台立刻啟動
乘客不用等，直接換台電梯繼續使用
```

### **常見的 FHRP 技術**
| 技術名稱 | 特色 | 白話說明 |
|---------|------|----------|
| **HSRP** | Cisco 專用 | 思科設備專用的備胎系統[2] |
| **VRRP** | 國際標準 | 各廠牌都能用的備胎系統[2] |
| **GLBP** | 負載平衡 | 兩台都同時工作，分擔流量[2] |

## **SVL (StackWise Virtual) - 堆疊虛擬技術**

### **白話解釋：兩台交換器「合體」成一台**
**就像把兩台電腦合併成一台超級電腦**，兩台實體交換器透過高速連線，對外表現得像一台設備。[3][4]

#### **實際運作方式**
- **物理上**：兩台獨立的交換器
- **邏輯上**：系統把它們當成一台來管理
- **連接方式**：用 10G 或 40G 高速線路連接[5]
- **管理介面**：只需要登入一次就能管理兩台[3]

#### **生活化比喻**
```
就像雙胞胎兄弟開店
外人看起來是一家店（一個老闆）
實際上是兩個人在分工合作
如果其中一個人請假，另一個人可以獨自撐場面
```

### **SVL 的組成要素**
- **SVL Link**：兩台交換器間的高速連線，就像雙胞胎的「心電感應」[6]
- **DAD Link**：偵測是否有分裂腦問題的檢查線路[6]
- **Domain**：給這個「雙胞胎組合」一個身分證號碼[6]

## **BGP (Border Gateway Protocol) - 邊界閘道器協定**

### **白話解釋：網際網路的「郵政系統」**
**就像國際郵件系統**，負責決定資料封包要走哪條路線才能到達目的地。[7]

#### **實際運作方式**
- **用途**：連接不同的網路系統（像不同國家的郵政系統）
- **工作方式**：各網路系統互相交換路由資訊[7]
- **傳輸方式**：使用 TCP 169 埠進行通訊[7]
- **更新機制**：定期傳送「我還活著」的訊號（預設 60 秒）[7]

#### **生活化比喻**
```
就像國際快遞公司的配送系統
FedEx 要送包裹到台灣，會跟台灣郵局合作
各家快遞公司會互相分享：
「我負責美國區域」「我負責歐洲區域」
這樣包裹就知道該走哪條路線最快到達
```

### **BGP 的兩種類型**
| 類型 | 英文縮寫 | 白話說明 |
|------|----------|----------|
| **內部 BGP** | iBGP | 同公司內部各分公司間的溝通[7] |
| **外部 BGP** | eBGP | 不同公司間的溝通協調[7] |

## **為什麼這些技術都很重要？**

### **企業網路的三大需求**
1. **高可用性**：FHRP 確保網路不斷線
2. **高效能**：SVL 提供更大的處理能力  
3. **全球連通**：BGP 讓資料能在全世界流通

### **在 Nexus Dashboard 中的角色**
- **FHRP**：確保管理介面的高可用性
- **SVL**：讓多台設備協同工作
- **BGP**：處理與其他網路系統的連接

這些技術組合起來，就像搭建一個**既穩定又高效的企業網路基礎設施**，確保公司的網路服務 24 小時不中斷。

## **網路架構進階概念**

### **LWAPP (Lightweight AP Protocol)**
**白話解釋**：**輕量級 AP 的通訊協定**
- **Layer 2 模式**：AP 和控制器在同一網段
- **Layer 3 模式**：可跨網段運作，適合大型環境
- **用途**：讓 Thin AP 與 Wireless Controller 溝通[4]

### **Distribution System**
**白話解釋**：**無線流量進入有線網路的橋樑**
- **功能**：將無線客戶端的流量導入有線網路環境

# CCNA 認證完整範圍解析

## **CCNA 是什麼？**

**CCNA (Cisco Certified Network Associate)** 是思科的**入門級網路工程師認證**，可以說是進入網路領域的「駕照考試」。[1][2]

- **CCNA**：學習**基礎網路概念和操作技能**
- **NDFC**：企業級的**進階網路管理平台**
- **關係**：CCNA 是學會「開車」，NDFC 是學會操作「大型運輸車隊管理系統」

## **CCNA 六大核心範圍**

### **1. Network Fundamentals (網路基礎)**
**白話解釋：學會「網路是怎麼運作的」**

| 學習內容 | 實際應用 | 就像學習 |
|---------|----------|----------|
| **OSI 七層模型** | 理解資料如何在網路傳輸 | 郵件系統的運作流程 |
| **TCP/IP 協定** | 了解網際網路的語言 | 學習國際通用語言 |
| **IP 位址與子網路** | 設定網路設備的「門牌號碼」 | 城市規劃與地址分配 |
| **網路設備認識** | 路由器、交換器、防火牆功能 | 認識交通系統的各種設施 |

### **2. Network Access (網路存取)**
**白話解釋：學會「讓設備互相連接和溝通」**

- **VLAN 設定**：就像把辦公室分成不同部門，各部門有各自的網路[3][1]
- **交換器配置**：設定網路設備讓電腦可以互相傳資料
- **無線網路基礎**：WiFi 的設定和管理[2]

### **3. IP Connectivity (IP 連線)**
**白話解釋：學會「資料如何找到正確的路徑」**

- **路由概念**：就像 GPS 導航，決定資料要走哪條路
- **路由協定 (OSPF、EIGRP)**：讓路由器自動找到最佳路徑[3]
- **第一跳冗餘協定 (HSRP)**：備胎機制，主要路由器壞了有備用的[3]

### **4. IP Services (IP 服務)**
**白話解釋：學會「網路的各種便民服務」**

| 服務類型 | 功能說明 | 生活化比喻 |

| **DHCP** | 自動分配 IP 位址 | 飯店自動分配房間號碼 |
| **DNS** | 將網址轉換成 IP 位址 | 電話簿查號服務 |
| **NAT** | 私人 IP 轉公共 IP | 公司總機轉分機 |
| **NTP** | 時間同步服務 | 校正所有時鐘到標準時間 |

### **5. Security Fundamentals (安全基礎)**
**白話解釋：學會「保護網路不被壞人入侵」**

- **Access Control List (ACL)**：設定「誰可以進入、誰不能進入」的規則[1][3]
- **VPN 基礎**：建立安全的遠端連線通道
- **基礎防火牆**：網路的「保全系統」
- **密碼政策**：設定強密碼和安全措施[3]

### **6. Automation and Programmability (自動化與可程式化)**
**白話解釋：學會「用程式自動管理網路」**

- **Python 基礎**：用程式語言自動化網路管理[2]
- **SDN 概念**：軟體定義網路，像 Nexus Dashboard 這樣的系統[1][3]
- **REST API**：讓不同系統可以互相溝通的介面[3]
