## 🔧 FortiGate SD-WAN無中斷遷移方案

### 📋 當前環境分析（IP已隱私處理）

**WAN介面狀態**：
- **wan1**: IP `10.1.1.49/24`（主要連線）
- **wan2**: DHCP模式（備用連線）  
- **internal**: IP `192.168.100.1/24`

**VPN隧道數量**: 7條主要隧道
**政策數量**: 約20條防火牆政策
**流量狀況**: 主要流量集中在wan1介面（4.27TB）

## ⚠️ 關鍵風險評估

### **高風險項目**
1. **VPN隧道中斷** - 多條商業關鍵隧道正在運行
2. **BGP路由** - SSL VPN連接Azure的BGP配置
3. **NAT政策** - 大量依賴wan1的NAT規則

### **建議遷移時間**: 業務低峰期（例如：凌晨2-4點）

## 🚀 無中斷遷移步驟

### **階段一：準備工作（無中斷）**

#### 1. 建立完整備份
```bash
# CLI執行
execute backup config management-station comment "Pre-SDWAN-Migration"
```

#### 2. 創建SD-WAN Zone（先不加入介面）
```bash
config system sdwan
    set status enable
    config zone
        edit "wan-zone"
            set comment "Primary WAN Zone for SD-WAN"
        next
        edit "wan-backup-zone"  
            set comment "Backup WAN Zone"
        next
    end
end
```

#### 3. 預先建立Performance SLA
```bash
config system sdwan
    config health-check
        edit "wan1-health"
            set server "8.8.8.8"
            set protocol ping
            set interval 1000
            set probe-timeout 1000
            set recoverytime 10
        next
        edit "wan2-health"
            set server "1.1.1.1" 
            set protocol ping
            set interval 1000
            set probe-timeout 1000
            set recoverytime 10
        next
    end
end
```

### **階段二：準備遷移用的暫時政策**

#### 4. 建立臨時Dummy介面
```bash
config system interface
    edit "dummy-wan"
        set vdom "root"
        set type loopback
        set ip 127.0.0.2 255.255.255.255
    next
end
```

#### 5. 批量修改防火牆政策（使用CLI較快）
```bash
# 備份當前政策到文字檔案
show firewall policy > current-policies.txt

# 暫時將wan1, wan2的政策指向dummy介面
config firewall policy
    edit 1
        set srcintf "internal" 
        set dstintf "dummy-wan"  # 暫時使用
    next
end
```

### **階段三：執行SD-WAN遷移（約5分鐘中斷）**

#### 6. 加入WAN介面到SD-WAN
```bash
config system sdwan
    config members
        edit 1
            set interface "wan1"
            set zone "wan-zone"
            set gateway 10.1.1.1  # 隱私處理後的閘道
            set source 10.1.1.49
            set cost 10
            set priority 1
        next
        edit 2
            set interface "wan2"  
            set zone "wan-backup-zone"
            set cost 20
            set priority 2
        next
    end
end
```

#### 7. 建立SD-WAN規則
```bash
config system sdwan
    config service
        edit 1
            set name "Primary-Route"
            set mode priority  
            set dst "all"
            config sla
                edit "wan1-health"
                next
            end
            set priority-members 1 2
        next
        edit 2
            set name "VPN-Traffic"
            set mode priority
            set dst "all"
            set priority-members 1  # VPN只走主線避免非對稱路由
        next
    end
end
```

#### 8. 更新路由表
```bash
# 刪除原有預設路由
config router static
    delete 1  # 假設原預設路由ID為1
end

# 新增SD-WAN預設路由  
config router static
    edit 1
        set device "wan-zone"
        set distance 1
        set comment "SDWAN Default Route"
    next
end
```

### **階段四：恢復政策並驗證**

#### 9. 將防火牆政策指向SD-WAN Zone
```bash
config firewall policy
    edit 1
        set srcintf "internal"
        set dstintf "wan-zone"  # 從dummy改為SD-WAN zone
    next
    # 重複其他政策...
end
```

#### 10. 特殊處理VPN政策
```bash
# 對於BGP VPN隧道，確保只使用wan1
config firewall policy  
    edit 15  # toHQ政策
        set srcintf "internal"
        set dstintf "wan-zone"
        # 在SD-WAN規則中確保VPN流量只走主線
    next
end
```
